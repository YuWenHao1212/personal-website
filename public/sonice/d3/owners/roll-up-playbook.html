BLOCK 3 · 案例二 · 你這塊的 playbook

roll-up 全公司戰情室 · 最後合體

你這塊 · Carol · 李總　｜　SO NICE 戰情室

目標做出來大概長這樣（你可以加減）

把七個人的戰情室收成一張 —— 李總打開一頁，公司總業績 + 五業務紅綠燈一眼看完，點一塊就鑽進那個人的 dashboard。
其他組（FB · 品類）一條連結直達 公司總業績 = 線上 ＋ 門市 ＋ 蝦皮（三通路相加） 五業務卡（達成 / 線上 / 蝦皮 / 庫存 / 競品）+ YoY 紅綠燈 點一張卡 → 鑽進該負責人 dashboard

這次怎麼做你這塊跟別人不一樣

別人顧一張表，你是把大家收成李總一頁看全公司。今天親手建一次 —— 之後換真資料、加新組你都做得到。先花一分鐘看懂它怎麼運作：

先懂架構李總一頁看全公司 —— 靠兩個「共用的地方」

七個人各自做自己的儀表板、各自上線到自己的網址。李總要一頁看完 —— 不是叫七個儀表板互相對話，是大家把東西放到兩個共用的地方，你的 roll-up 去那兩邊撿。
① 資料（數字）
流量 / 達成 / 蝦皮 / 庫存 的 CSV，每份只有自己那條通路。公司總業績沒寫在任何一份裡 —— 是你把線上＋門市＋蝦皮 三份加起來才生出來的。整合就發生在這層。
② 公佈欄（各組門牌 · 競品情報）
一塊共用白板。每個人部署完就把自己的網址貼上去。你讀它 → 知道每組 dashboard 在哪、點卡連去哪；競品的即時資料也掛在這。
你的 roll-up = 讀 ① 算出公司總業績＋五業務卡，讀 ② 讓每張卡點得進那個人的 dashboard、競品資料也跟著進來。它只是個讀取者，把兩邊撿來拼成一頁。
公佈欄是老師課前架好的後端 —— 網址在你的 戰情室設定.txt 裡（步驟 1 下載），你是拿到、不是自己寫。競品的即時資料也掛在公佈欄上（家喻的爬蟲），roll-up 跟著它讀，你不用另外填。

操作步驟照著走

  1. 下載你的材料 各組資料 ＋ 品牌 ＋ 公佈欄設定
    收齊各組的資料（線上 / 門市 / 蝦皮 / 庫存）＋ 老師給的公佈欄設定。
    ⬇ 流量.csv（線上） ⬇ 達成.csv（門市） ⬇ 蝦皮.csv ⬇ 庫存-商品.csv ⬇ 戰情室設定（公佈欄網址）
  2. 開專案、把檔案收進去
    把下面這句照貼給 Claude Code：
    在 Efforts/Projects/Active 開一個新專案叫 Demo_李總戰情室，把我下載的 流量.csv、達成.csv、蝦皮.csv、庫存-商品.csv 和 戰情室設定.txt 搬進去，之後在裡面做事。
  3. 研究 先別做
    讓 Claude 先看懂這幾份表怎麼相加 —— 先別動手：
    研究這幾份 CSV：流量.csv 是線上（有 2026 和 2025 同週資料，直接比 YoY）、達成.csv 是門市 46 家當週快照（沒有週次欄，去年同週業績用「當週營業額 ÷ (1 + 去年同週成長率)」反推）、蝦皮.csv 是蝦皮（用實際業績欄、不用接單業績）、庫存-商品.csv 是庫存。公司總業績 = 線上營業額 ＋ 門市 46 家當週加總 ＋ 蝦皮實際業績，三通路不重疊。線上沒有目標欄，線上卡只顯示 YoY、不顯示達成率。每張卡只給數字和紅綠燈、不要解釋漲跌原因，原因留給我現場講、不確定就說不知道。先別動手做，先跟我講你打算怎麼算、這張板該放哪些關鍵數字。
  4. 討論 你最懂李總要看什麼
    Claude 提案後，用李總的眼光回它 —— 他每週第一眼要看的、紅綠燈怎麼定。
    你可以這樣說
    「公司總業績是我第一眼，其他都輔助」「輸去年同週就紅 —— 賠錢、衰退我要馬上看到」
  5. 做第一版 公司總覽 · 用 CSV 馬上能做
    不用等別人 —— 你手上的 CSV 就能把整張公司總覽做出來：
    好，照我們討論的，做一頁李總戰情室 HTML，讀同資料夾那幾份 CSV。最上面放「公司總業績」= 線上 ＋ 門市 ＋ 蝦皮 三通路相加、跟去年同週比 YoY，旁邊用長條顯示三通路各占多少。下面五張業務卡：門市達成 / 線上 / 蝦皮 / 庫存 / 競品，每張一個關鍵數字 + 紅綠燈（照我們剛剛討論的規則）。版面一次擺到位、外部資料先留空：競品卡先顯示「—」、五張卡先做成「之後可以點」的樣子、下面預留一條「其他組戰報：FB · 品類」連結列 —— 這些等下接公佈欄再填，現在先只讀本地 CSV、不連任何外部網址。外觀套 SO NICE 品牌風格。這頁等下要放上 Cloudflare Pages 給大家看，資料 CSV 會跟著一起放上去、頁面直接讀它 —— 別做那種要先開個伺服器、一直掛著才看得到的。做完直接幫我用瀏覽器打開。
  6. 接公佈欄 把各組網址 + 競品數字灌進預留的位置 · 合體的關鍵
    上一步把位置都留好了，這步只接真資料 —— 卡片變得點得進去、FB／品類 連得到、競品數字進來：
    上一步已經把卡片、FB／品類 連結列、競品卡的位置都留好了。現在接公佈欄、把真資料灌進去：去讀 戰情室設定.txt 裡那個公佈欄網址 —— 它告訴你各組 dashboard 的網址、競品資料掛在哪（你先讀讀看內容）。① 把五張卡和「其他組戰報：FB · 品類」那條連結，填上各組真實網址、點了開新分頁；② 競品卡的「—」換成公佈欄指到的競品資料。只填資料、不動版面，也不要做交件追蹤或刷新 —— 七組都已經做完了，這是最後合體的總覽、不是監控板。
  7. 微調 改到李總順眼
    它做一版、你看、說哪裡改 —— 來回幾次到順眼為止。
    你可以這樣說
    「公司總業績數字再大一點」「這張卡紅綠燈反了，本週低於去年才紅」「交件看板放最上面、最顯眼」
  8. deploy 上線
    用你下課裝好的 wrangler，把整個資料夾部署到 Cloudflare：
    幫我用 wrangler 把這個資料夾部署到 Cloudflare Pages。第一次建一個專案，名字用 sonice-warroom（全球不能撞名，撞名就加數字）。建好就部署，弄好直接幫我打開網址、把網址貼給我。

最後合體驗收

這張就是李總的板 —— 不用回報給誰。七組都部署、登記到公佈欄之後，把 roll-up 部署上線、投上大螢幕。李總打開這一頁：最上面公司總業績＋三通路占比，下面五張業務卡紅綠燈一眼掃完，點任一張就鑽進那一組的 dashboard 看細節。一頁 = 全公司看完。
課後要繼續用：把這幾份 CSV 換成每週真實匯出（欄位名不動），板自己更新；要加新一組，跟 Claude 說一句就好。正式版可把資料源搬到公司治理的儲存（有權限、有稽核）。
卡關？把錯誤訊息整段貼回去給 Claude —— 它會幫你修。真的卡住就舉手，老師過去看。
提醒：要從有這幾份 CSV 的資料夾、用 http 開啟（Claude 部署 / 開瀏覽器時會處理好），直接雙擊檔案會讀不到資料源。
