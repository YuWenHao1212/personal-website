BLOCK 3 · 案例二 · 你這塊的 playbook（第 1 步）

競品 · 讓它自己每週跑 第 1 步

你這塊 · 家喻　｜　SO NICE 戰情室
競品這塊比別人多一步 —— 你剛在電腦上做好的那支爬蟲，這一步把它搬上雲端：不用你開電腦，它也會自己跑、每週自動更新。做好會給你一個資料網址，下一頁戰情室（競品.html）讀它。

先搞懂這一步在做什麼 —— 把爬蟲變成一台自己運轉的小機器

案例一：你在電腦上「按一次、抓一次」—— 證明抓得到，但電腦一關就停。
這一步：同一支爬蟲，加兩個零件、送上雲端，變成一台自己運轉的小機器 ——
· 爬蟲（你已經做好的）—— 去對手網站抓本週新品
· 倉庫（資料庫）—— 抓到就存起來
· 鬧鐘（排程）—— 每週一自動叫它跑一次，不用你動手
做好後它開兩個出口：一個你打開、點一下就抓／看結果的頁（試跑用）、一個給下一頁戰情室讀的資料網址。
所以你不是再做一個漂亮展示頁（那是下一頁的新品牆）—— 你是幫爬蟲裝上倉庫和鬧鐘、送上雲端。

操作步驟在你案例一做爬蟲的那個 Claude 裡接著做

  1. 做出來、直接放上 Cloudflare 照貼
    在你案例一做爬蟲的那個 Claude Code 裡，照貼下面這段（白話講需求 ＋ 直接上線，怎麼做交給它）：
    把我剛剛抓 N.ONE 新品的程式，升級成「自己每週跑」、直接放上我登入的 Cloudflare： · 每週一自己去抓這禮拜的新品（每件一張封面圖，跟案例一一樣），不用我手動 · 抓到就存起來；被擋就用上次存的、不開天窗 · 給我一個資料網址，把整理好的資料吐出來，給我下一頁的戰情室讀 · 再給我一個用瀏覽器打開、點一下就「現在就抓一次」的頁面（我不會打指令、別用密碼鎖） 這隻是背景抓資料用的、不用漂亮（展示牆下一頁做）。只用 Cloudflare 一個平台（我登入的那個），別牽扯 GitHub 或其他服務；我不是工程師，密鑰越少越好。要建、要設定的，你直接用 wrangler 處理好。弄好清楚告訴我哪個網址「點了就抓」、哪個「貼去登記給戰情室讀」。
  2. 先試跑一次 關鍵 · 別跳過
    打開 Claude 給你的「現在就抓」那一頁（它做完會告訴你是哪個網址）→ 按一下 → 等它抓到本週新品（十幾到三十件都正常）。
    為什麼一定要先按這一次
    確認它真的抓得到（不是空的）。
  3. 登記你的資料網址 填一次、自動帶
    Claude 做完會給你好幾個網址 —— 你要貼的是它標明「給戰情室讀 / 新品資料」的那一個（不是「點了就抓」那個、也不是看狀態的）。貼進來按「登記」—— 下一頁戰情室會自動帶、李總那張也會改讀你這隻。
