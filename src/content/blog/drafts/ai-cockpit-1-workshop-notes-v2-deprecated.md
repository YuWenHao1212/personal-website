5/16 早上 9 點，台北，12 個學員坐定，我打開 deck 第一張投影片。

深呼吸。

這是 AI Cockpit 這個名字第一次出現在正式工作坊的場合。在那之前，這個名字只存在我自己的 Obsidian vault、幾個朋友的 LINE 對話、還有三月底一次線上課前 session 的 Zoom 視窗裡。

但 5/16 那天工作坊上要教的東西，不是 5/16 才開始的。

是我用 Claude Code 將近一年、慢慢演化出來的一套結構。後來我叫它 FLUX。今天是 FLUX 第一次被架構化地教出去。

我知道 4 小時能改變的很有限。一套我自己用了半年才長出來的系統，要在一個早上讓 12 個人從零開始理解它，這件事本身就有點荒謬。

但我還是決定做。原因很簡單：我已經有了來自其中一位學員的反饋。那份反饋比我自己的任何 deck 都精準，而且他給我那句話的時間，是課前 1.5 個月。

這篇文章是 5/16 的紀錄，從學員那裡學到的事 —— 不是教學筆記。

---

## Claude Code 之前：6-7 個工具、0 個系統

2025 年 5 月以前，我的工作工具清單大概長這樣：Notion 記任務、Obsidian 塞讀書筆記、Apple Notes 隨手丟想法、Google Docs 寫草稿、ChatGPT 問問題、再加上各種 todo app 輪流試然後輪流棄。

6-7 個工具。都在跑。都有進東西。但沒有一個能告訴我：我昨天做了什麼、我這個月的目標推進到哪裡、我三個月前讀的那篇文章跟現在在做的事有沒有關係。

那些工具各自活在各自的世界。我是唯一一個在它們之間跑來跑去的人。

有一個下午，投資人問我：「你最近在做什麼？可以讓我看看嗎？」

我打開電腦。

Notion 有任務清單，但沒有脈絡。Obsidian 有一堆筆記，但找不到。Google Docs 有幾份草稿，但版本亂掉了。我滑了 5 分鐘，找不到一個可以「給他看」的東西。

那個下午我沒有拿出任何東西。

努力夠了。缺的是把努力顯示出來的地方。

那時候我其實心裡有個隱隱的想法：不知道是工具的問題還是我的問題。我懷疑是我的問題。後來才知道兩個都是，但問錯問題了。我一直在問「哪個工具最好用」 —— 該問的是「有沒有一個系統把所有工具串起來」。

2025 年 5 月，Claude Code 正式上線。

老實說，頭幾個月我也只是把它當另一個 AI 聊天工具在用。請它寫程式、請它幫我潤稿、請它回答問題。它確實比其他 AI 工具好用，但我還沒搞清楚「好用」在哪裡。

大概用了半年之後，我才真正理解它的本質。

Claude Code 沒有形狀。它沒有預設介面、沒有預設工作流程、沒有預設的「你應該這樣用」。這在大部分軟體工具裡是缺點——沒有 onboarding、沒有 tutorial、沒有範本。

但我後來發現，沒有形狀代表它裝得進任何形狀。

我把 Obsidian vault 接進去。把 Google Calendar 接進去。把 Gmail 接進去。把我的 12 週目標接進去。把我的日誌習慣接進去。每接一個，系統就多長出一節。半年後，那個東西不再是一堆工具的合集，它開始有自己的邏輯了。

那時候我給它取了名字。我叫它 FLUX。

再過半年，FLUX 長出了一個我可以教的形狀。5/16 那天，我把它第一次架構化地教出去。

## FLUX 是什麼、為什麼是這個形狀

設計者視角解釋一套系統，最難的地方是要忍住不講太多。

我來嘗試只講三件事。

**FLUX 是 4 個字母**：F / L / U / X，對應 Focus / Learn / Upgrade / eXecute。

我選 4 個不是 5 個，是因為 5 個記不住。選 4 個不是 3 個，是因為 3 個包不住（3 個永遠有一個面向不知道塞哪裡）。4 個在我測試的各種個人工作框架裡，剛好是「記得住又夠用」的邊界。

但名字其實是次要的。比名字更重要的是 —— FLUX 是 loop，不是 list。

List 永遠做不完。To-do list 在語義上是「等待被劃掉的清單」，永遠有做不完的感覺。FLUX 的「做了 → 學了 → 升級了 → 再執行」是一個循環。Loop 不會做不完，它只是在轉。

**FLUX 有 3 層**。不是技術文件裡的 UPPER / MIDDLE / FOUNDATION，用人話說：

最上面那層：我這輩子要做什麼。12 週目標、人生方向、長期方向感。

中間那層：我今天怎麼做。Daily note、週回顧、任務排序、今天的優先事項。

最下面那層：我用什麼裝這些。Claude Code 的 CLAUDE.md、skills、vault 架構、工具串接。

為什麼 3 層不能是 2 層？因為少了中間那層，上下兩端會斷掉。人生目標跟今天的任務清單之間，需要一個轉換機制——「12 週目標 → 本週計畫 → 今天任務」這個轉換不是自動發生的，需要設計。

我在 deck 裡寫了一句話：「底座沒打好，上面都會歪。」那句話原本是寫給學員的。課上完、回頭看，我發現那也是在寫我自己——我的「底座」是在一年的摸索裡才慢慢打好的，不是設計好的。

**FLUX 有一個我還沒想清楚的地方**。

在個人 vault 裡，Area（永續經營的領域）和 Project（有結束點的專案）的邊界很清楚。但如果有人要在團隊裡用 FLUX，多人共用同一個 vault，Area 和 Project 的界線就開始模糊了。這個問題我在 5/16 沒有好答案，現在也沒有。

這是 v1 的侷限。我不想假裝沒有。

這個結構我自己用了大概半年才長出來。但 5/16 那天我有 4 小時把它教給 12 個人。教之前的 1.5 個月，我已經有一份來自其中一位的反饋。他叫 Jay。

## 三個學員、三句話 —— 但其中一位用了 1.5 個月

### Jay：提前了 1.5 個月

AI Cockpit #1 有個設計：12 位學員每人都有一次 1 對 1 課前 session。標配 —— 沒有附加費用、沒有選配。60 分鐘，我們一起把環境裝起來、把 CLAUDE.md 寫進去、設定 12 週目標、跑一次 daily 和 WAM 的完整流程示範。

課前 session 的概念是：讓每個人帶著一個「已經動過一次的系統」來上課，而不是對著一個全陌生的東西坐著。

Jay 的 session 是 3/31，線上，1 小時。

Jay 是軟體業的工程主管。他看了我寫的文章，然後報名。那次 session 我們做了該做的事，session 最後我說了一句：「卡住就 LINE 我。」

然後，我們沒有再聯絡。

1.5 個月。

5/16 開課前我問他：「這段時間用得怎樣？」

他回了我一句：「從人生目標到每天任務管理，透過 FLUX 框架和 Claude Code 協作 —— 整條路徑變得清晰、有方向感。」

我把那句話讀了兩遍。

我不知道 Jay 這 1.5 個月每天跑了什麼指令。不知道他的 12 週目標完成度。不知道他有沒有每週確實做 WAM。那些我完全不知道。

但他用一句話描述了我的系統。而那句話比我自己 deck 上寫的任何一句都精準。

「路徑清晰」、「方向感」——我自己的 deck 裡從來沒用過這兩個詞。

我的 deck 講的是「個人指揮中心」「FLUX 四層架構」「daily skill 流程」。那些是設計者的語言——精確但冷。Jay 給我的是使用者的語言——不精確，但是對的。

3/31 那天我教 Jay 的時候，我是在教一個學員。

5/16 之後我才意識到，Jay 也教了我一件事——一套系統最後的價值不是效率，是方向感。

### Yiling：下課之後她寫了一段話

Yiling 的課後反饋裡有幾句話讓我多看了幾次。

「過去常被截止日期追著跑，總是很焦慮自己事情做不完。」

這個感覺我自己也有過，而且有很長一段時間。然後她寫：

「我成為一個有私人秘書的總裁級人物。」

「我還會有一個私人的圖書館以及圖書館員。」

我的 deck 上講的是「個人指揮中心」——這是一個技術用語，是結構性的命名。Yiling 翻出來的是「私人秘書」加「圖書館員」——那是生活用語，是體感的命名。

我的命名給自己用方向。她的命名讓沒進過 vault、沒碰過 Claude Code 的人也接得住。

一套系統需要兩種命名才完整：設計者的命名、使用者的命名。我只能負責前者。

但 Yiling 不是下課之後想出這兩個詞的。

她是在 5/16 下午的某一個瞬間，看見了什麼，然後在心裡有了這個詞，下課才寫下來。

那個瞬間發生在 Block 4。

### 延宗：方法論的翻譯

延宗的課後反饋是用方法論語言寫的。

「從工具思維升級至系統化流程。」「將目標、知識、任務與輸出整合成完整的閉環。」「真正將 AI 轉化為長期複利與實質的產能提升。」

（「閉環」、「複利」是延宗自己的用詞 —— 我平常寫 loop、寫累積。下面引用他的話我都保留原貌。）

三句話。每一句都是方法論的概念，但用的是他自己的語言框出來的。

「閉環」、「複利」——我的 deck 裡也沒有這兩個詞。我當然知道這兩個詞。設計框架的時候，我選了「loop」、選了「累積」，沒有選「閉環」和「複利」。

但延宗的翻譯是對的。他看清楚了 FLUX 想做的事，然後用他自己的語言說了出來。

Jay 給了我「方向感」。Yiling 給了我「私人秘書」。延宗給了我「閉環」和「複利」。三個人用完全不同的語言，講的是同一件事——我設計的這個東西真的是一個系統，不是 4 個 AI 工具的合集。

但延宗說的「閉環」要成立，4 個 Block 怎麼排、為什麼這樣排、為什麼某個東西要等到 Block 4 才出現——就變成這篇紀錄不能跳過的事。

## 5/16 那天 4 個 Block 是怎麼跑的

### Block 0：兩個早上

開場投影片我放了一個對比：兩個早上。

第一個早上：起床，打開電腦，不知道今天要先做什麼。看一眼 Email，打開 Notion，滑一下 IG，打開另一個 tab，然後在 10 點意識到自己還沒真正開始。

第二個早上：打一行「開工」，Claude Code 讀今天的日曆、讀你寫進去的目標、讀昨天的紀錄，把今天的優先事項清單給出來。挑一個，開始做。

設計這張投影片的時候我猶豫過——要用故事開場還是用 framework 開場？故事讓人有畫面，但容易讓人以為「這是個人經驗分享」。Framework 清楚，但進不了任何人的身體感覺。

我選了故事。

因為如果第一張投影片就講 FLUX 是什麼，大部分人會把它當一個技術架構來聽，然後很快決定「這跟我沒關係」。先讓他們看見那兩個早上的差別，再講這個差別是怎麼來的——那個順序有意義。

### Block 1：認識 Claude Code

75 分鐘。

這一節講了一件在 2025 年 5 月之前幾乎所有人都沒辦法直觀理解的事：Claude Code 跟 Claude.ai 是兩種不同的工具，儘管背後是同一家公司的模型。

Claude.ai 是對話 —— 使用者問它，它回答。Claude Code 是代理人 —— 使用者給它工具和目標，它去做事。

中間有個分野。屬於使用哲學的層次，不是技術差異。

我在這節說了一句話：「Claude Code 沒有形狀，所以它裝得進任何形狀。」

然後我讓每個學員用 Claude Code 說一次自己的名字。

不是什麼複雜的指令。就是「你好，我是 [名字]，請跟我打招呼」。

那個當下，每個人第一次用自己的鍵盤、對一個模型說出自己的名字，然後看著它回應。

那個時刻有點小，但我在每個課前 session 都做這件事。讓人跟一個工具建立第一次接觸，比任何 tutorial 都有效。

### Block 2：目標管理

12 Week Year + Lead vs Lag 指標。

這節最技術。但也是最多人課後記得的一節。

我在 Block 2 講了那個投資人的下午——我打開電腦、卻拿不出任何東西給他看。講的時候我沒有加戲劇效果。就是把那件事說出來：有人問我「你最近在做什麼」，我找不到一個能給他看的東西。

那段講完之後，現場有一段短暫的安靜。

然後 pair-share 開始了。我請每組兩個人互相說自己的 12 週目標。那 8 分鐘是全天現場最熱的 8 分鐘——不是因為我設計了什麼活動，是因為大部分人很少被問「你接下來 12 週的目標是什麼，說出來給我聽」。

延宗後來說的「閉環」就是從這節開始有了形狀。目標不只是設下來，要設 Lead 指標，要每週回顧，要跟 daily 接起來。那個接法——目標 → 週計畫 → 每日 → 回顧 → 目標——才是閉環。

### Block 3：知識管理

LYT 框架、Atlas 結構、knowledge-keeper Skill。

我沒有在這節展開 Skill 怎麼寫。那太技術了，跟學員當下的需求離得有點遠。我把焦點放在結果：讀過的東西不需要靠人記得，系統會在需要的時候把它找出來。

我說了一個比喻：大部分人的知識管理是「倉庫」——東西進去了，但沒有人整理，找起來靠運氣。FLUX 的知識層要做的是「有圖書館員的圖書館」——索引清楚、知道放哪、需要的時候能找到。

Yiling 後來寫的「圖書館員」就是從這節長出來的。她接住了那個比喻，然後把它帶回到自己身上。

### Block 4：串接＋自動化

這是 Yiling 看見她看見的東西的那一節。

Gmail 串接。Google Calendar 串接。兩個設定裡的 toggle，打開。OAuth 點兩下，授權。

然後我請每個學員在 Claude Code 裡打一個字：「開工」。

那個當下，Claude Code 第一次讀到她們真實的會議——今天下午 3 點有一個 Zoom、明天早上要交一份提案。第一次讀到真實的信件——老闆昨天寄了一封還沒回、廠商在等確認。第一次讀到本機存著的、她們親手打進去的 12 週目標。

Daily note 自己長出來了。今天的優先事項清單。

然後 Yiling 喊出來：「Claude Code 真的回覆我行事曆上的行程了耶！」

我那一刻沒講話。

因為我知道，接下來幾個小時之內，她會自己想出一個詞來形容她剛剛看見的東西。她下課寫下的那個詞是「私人秘書」。

一個系統最好的證據，是使用者用自己的語言重新命名它。

## 收尾

5/16 下午 4 點，12 個學員一個個關上電腦，站起來，離開。

我把投影機關掉，把椅子歸位，看著空的會議室。

想起 Jay 說的「方向感」。想起 Yiling 喊的「真的回覆我行事曆了耶」，然後下課寫的「私人秘書」。想起延宗寫的「複利」。

FLUX 我自己用半年才長出來。5/16 那天有 4 小時、12 個人從零開始看見它。然後三個人用三種完全不同的語言重新命名了它。

我終於可以說——這套東西不只對我自己 work。

但 5/16 結束後我也知道，這只是 v1。FLUX 還會改，deck 還會改，4 個 Block 的設計還會改。下次有人來上課，會是另外一個版本的我，教另外一個版本的這套東西。
